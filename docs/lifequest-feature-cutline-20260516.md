# Life Quest Feature Cutline Plan - 2026-05-16

## 목적

현재 Life Quest 기능을 출시 리메이크 관점에서 `유지`, `재배치`, `숨김`, `삭제 후보`, `장기 보류`로 나눈다. 기준은 냉정하게 잡는다. 기능을 많이 보여주는 것이 목표가 아니라, 사용자가 첫 3분 안에 앱의 이유를 이해하고 다음 날 다시 돌아오게 만드는 것이 목표다.

## 판단 기준

각 기능은 아래 5개 질문을 통과해야 출시 1차 전면에 남긴다.

- 현실 행동과 직접 연결되는가?
- 첫 3분 안에 사용자가 이해할 수 있는가?
- 사용자가 다음 행동을 하게 만드는가?
- 출시 QA 범위 안에서 안정적으로 검증 가능한가?
- 유지율 또는 수익화에 기여하는가?

점수:

```text
2 = 강하게 해당
1 = 일부 해당
0 = 해당 없음 또는 오히려 방해
```

출시 판정:

```text
8~10점: 유지
5~7점: 재배치
3~4점: 숨김
0~2점: 삭제 후보
정책/권한/운영 리스크 큼: 장기 보류
```

## 출시 1차 화면 구조 컷라인

### 하단 탭 4개로 축소

현재:

```text
상태 / 퀘스트 / 던전 / 상점 / 업적
```

리메이크 후보:

```text
오늘 / 성장 / 모험 / 보상
```

### 탭별 포함 기능

#### 오늘

전면:
- 오늘의 모험 요약
- 일일 퀘스트
- 주간 퀘스트 일부
- 집중 타이머 진입
- 오늘 던전 보정 미리보기

숨김:
- 월간/연간 레이드
- 상세 리포트

#### 성장

전면:
- 레벨/XP
- 힘/지혜/건강/매력
- 현재 칭호
- 다음 칭호까지 남은 조건
- 최근 행동 기반 성향

숨김:
- 상세 스킬 트리
- 장비/인벤토리

#### 모험

전면:
- 소울 덱 시작
- 던전 1챕터
- 오늘 보정 표시
- 카드 보상
- 이벤트

숨김:
- 무한 타워
- 승천 모드
- 카드 컬렉션 전면 화면
- 유물 도감

#### 보상

전면:
- 현실 보상
- 커스텀 보상
- 골드 사용
- 업적 요약

숨김:
- 코스메틱 상점
- 수익화 안내

## 삭제 후보 상세

### 인벤토리 중심 구조

문제:
- 현실 행동 앱에서 인벤토리는 의미가 약하다.
- 장비/아이템이 없으면 빈 껍데기처럼 보인다.
- 상태창/성장 프로필과 겹친다.

결정:
- 출시 1차에서는 인벤토리 전면 진입을 제거한다.
- 필요한 수치만 성장 프로필로 흡수한다.

완료 조건:
- 하단 탭/주요 CTA에서 인벤토리 진입 제거
- 인벤토리의 유효 정보가 성장/보상/모험 중 하나로 이동
- 기존 저장 데이터는 삭제하지 않고 숨김

### 월간/연간 레이드

문제:
- 초반 사용자에게 너무 장기 목표다.
- "레이드"라는 표현이 앱 목적 이해를 방해할 수 있다.

결정:
- 데이터는 보존하되 출시 1차 UI에서는 숨긴다.
- 나중에 `장기 목표` 또는 `시즌 목표`로 재명명한다.

### 무한 타워/승천

문제:
- 하드코어 카드 게임 문법이라 초반 사용자에게 과하다.
- QA 범위를 폭발시킨다.

결정:
- 출시 1차 전면 금지.
- 내부 개발 플래그 또는 숨김 메뉴로만 유지.

## 보류 기능 조건

### Health Connect

공식 Health Connect 문서는 데이터 타입과 권한 계획을 먼저 이해해야 한다고 안내한다. 일부 데이터 타입은 특별 승인이 필요할 수 있다.

도입 조건:
- 개인정보 처리방침 업데이트
- Health apps declaration form 준비
- 사용자가 권한 제공 이유를 이해하는 UX
- 최소 1개 데이터 타입만 파일럿

1차 후보:
- Steps
- ExerciseSession
- SleepSession

금지:
- 출시 1차에 여러 건강 데이터 동시 요청
- 민감 건강 조언 자동 생성

### 구독

Google Play Billing 구독은 lifecycle, 갱신, 만료, 결제 실패, entitlement 처리가 필요하다. RevenueCat 2026 자료도 구독은 첫 갱신 전 가치 증명이 중요하다는 점을 보여준다.

도입 조건:
- D7/D30 retention 확인
- 프리미엄 가치 명확화
- 백엔드 entitlement 처리 준비

출시 1차:
- 구독 금지
- 광고 제거/후원/테마성 일회성 결제만 검토

## 완료 조건

- 기능 컷라인 표가 현재 주요 기능을 모두 포함한다.
- 출시 1차 탭 구조가 4개 이하로 정리된다.
- 숨김/삭제 후보에 대해 데이터 보존 방침이 있다.
- 다음 구현 문서인 Core Loop UX 상세 계획과 충돌하지 않는다.
