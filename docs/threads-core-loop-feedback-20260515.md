# Threads Core Loop Feedback - 2026-05-15

## 목적

Threads 테스터 모집 글에 달린 제품 방향성 피드백을 기록한다. 이번 피드백은 단순 UI 버그가 아니라 Life Quest의 핵심 루프, 정보 구조, 게임 목적의식, 현실 행동과 RPG 보상 연결 방식에 대한 지적이다.

## 원문 요지

### 1. 유명한 게임처럼 움직임과 이벤트를 체크할 필요가 있다

- 햄피드처럼 어느 정도 움직임을 체크하고, 이러한 갈림길 이벤트가 매번 변하는 범위가 넓어질 수도 있어야 한다.
- 예를 들어 인바디 프로필을 올리면 힘, 지능, 덱스, 체력, 매력 등으로 변환되어 나올 수도 있다.
- 아키텍처가 무엇인지부터 명확히 해야 한다.

### 2. 쿠팡 리셀러보다 리셀러 개념으로 접근해야 한다

- 재미없으면 누구도 안 한다.
- 타이틀을 얻으면 어떤 이벤트가 열릴지 고민해야 한다.
- 되도 않는 상태에서 테스트하려고 모아서 욕만 먹지 말라는 경고성 피드백이 있었다.

### 3. 라이프 퀘스트라면 현실 행동 데이터가 게임으로 해석되어야 한다

- 내 삶에 대한 라이프 퀘스트라면 이력서나 AR 연동 같은 방향으로 현실 기록과 연결되어야 한다.
- 예시:
  - 내가 글을 몇 개 발행했는지
  - 업로드를 하고 무엇을 분석했는지
  - 비타민/칼로리를 얼마나 섭취했는지
  - 러닝/운동/일의 실제 수행량
- 이런 현실 데이터를 기반으로 무엇을 권장하고, 어떤 음식이 안 좋은지, 무엇이 부족한지 분석해야 한다.
- 현재는 게임 목적의식이 약해 보인다.
- 이미 경쟁할 게임이 많으니 재미와 유지 동기가 없으면 안 된다.
- 내가 한 칼로리 소비, 수면, 운동, 행동 데이터가 스탯에 얼마나 반영되었는지 확인되어야 한다.
- 타이틀 명단계와 그에 연결된 이벤트가 제대로 열려야 한다.
- 보조 작품/공략 기반 추천처럼 외부 데이터와 행동 기록을 섞어 추천해주는 방향도 고려할 수 있다.

## 제품 관점 해석

이번 피드백의 핵심은 "기능이 많다"가 아니라 "왜 계속 써야 하는지"다.

현재 Life Quest는 퀘스트, 상태창, 던전, 카드, 보상, 상점, 칭호가 존재하지만 사용자가 첫 1분 안에 아래 흐름을 명확히 이해하기 어렵다.

```text
현실 행동
-> 퀘스트 완료
-> 성장/스탯 변화
-> 던전에서 체감되는 힘
-> 칭호/카드/이벤트 해금
-> 다시 현실 행동으로 돌아가는 동기
```

따라서 다음 단계의 핵심은 화면 수를 늘리는 것이 아니라 위 연결을 강하게 만드는 것이다.

## 문제 정의

### P0. Core Loop 명확성 부족

사용자가 앱에 들어왔을 때 "오늘 무엇을 하면 어떤 성장과 게임 보상이 생기는지"가 한눈에 보이지 않는다.

필요한 방향:
- 오늘 할 행동
- 그 행동으로 얻는 XP/골드/스탯
- 던전에서 받는 보정
- 해금될 수 있는 카드/칭호/이벤트

위 정보를 하나의 흐름으로 보여줘야 한다.

### P0. 현실 행동과 RPG 수치의 연결 약함

퀘스트 완료가 XP와 골드로 이어지지만, 어떤 행동이 어떤 스탯/전투력/카드에 영향을 주는지 직관성이 부족하다.

필요한 방향:
- 운동/건강 퀘스트 -> 전투 HP, 방어, 회복 이벤트
- 공부/지식 퀘스트 -> 마법, 카드 드로우, 지혜 이벤트
- 업무/생산성 퀘스트 -> 골드, 전술 카드, 상점 할인
- 사회/매력 퀘스트 -> 이벤트 선택지, 보상 협상, 칭호 해금

### P1. 칭호와 이벤트의 의미 부족

칭호가 단순 수집물이 아니라 이벤트와 플레이스타일을 바꾸는 장치가 되어야 한다.

필요한 방향:
- 특정 칭호 획득 시 전용 이벤트 해금
- 칭호별 던전 시작 보정
- 칭호별 카드 보상 확률 변화
- 칭호별 현실 행동 추천

### P1. 첫 진입 정보 구조 부족

기능이 먼저 보이고 목적은 사용자가 추론해야 한다. 테스터에게는 아래 메시지가 먼저 전달되어야 한다.

```text
Life Quest는 현실 행동을 RPG 성장으로 바꾸는 앱이다.
오늘의 퀘스트를 완료하면 나의 스탯이 오르고,
그 성장이 소울 덱 던전에서 카드/이벤트/보상으로 체감된다.
```

### P2. 외부 데이터/자동 기록 연동은 장기 과제

인바디, 운동량, 칼로리, 수면, 글 발행량, 업로드/분석 기록 같은 외부 데이터 연동은 강력하지만 초기 범위로 잡으면 과도하다.

단기적으로는 수동 입력과 퀘스트 카테고리 기반 변환부터 강화하고, 장기적으로 Health Connect, Google Fit, Notion/GitHub/블로그/문서 활동 연동 등을 검토한다.

## 후속 작업 제안

### 1단계. Core Loop UX 문서 작성

새 문서에서 아래를 먼저 확정한다.

- 현실 행동 카테고리
- 각 카테고리가 올리는 스탯
- 스탯이 던전에 주는 효과
- 칭호가 열어주는 이벤트
- 오늘 화면에서 보여줄 핵심 요약

### 2단계. 첫 화면 또는 퀘스트 홈 재구성

현재 화면을 전면 갈아엎기보다, "오늘의 모험" 요약 블록을 먼저 추가한다.

예시:

```text
오늘의 모험
퀘스트 2개 완료 -> 지혜 +1, 골드 +18
던전 보정: 마법 카드 보상 확률 +8%
다음 칭호: 루틴 입문자까지 2일
추천 행동: 25분 집중 1회
```

### 3단계. 칭호-이벤트 매핑 설계

칭호를 단순 뱃지가 아니라 이벤트 해금 조건으로 바꾼다.

예시:
- `루틴 입문자` -> 첫 캠프 이벤트
- `새벽 탐험가` -> 아침 전용 이벤트
- `지식 수집가` -> 마법 카드 보상 이벤트
- `꾸준한 전사` -> 방어/회복 이벤트

### 4단계. 현실 행동 -> 전투 효과 매핑

퀘스트 완료 카테고리에 따라 다음 던전에서 체감되는 효과를 준다.

예시:
- 건강 퀘스트 완료: 다음 전투 HP +5
- 지혜 퀘스트 완료: 첫 턴 카드 +1장
- 힘 퀘스트 완료: 공격 카드 피해 +1
- 매력 퀘스트 완료: 이벤트 선택지 하나 추가

## 당장 하지 않을 것

- Figma/Canva/Hyperframes 기반 전체 리디자인
- Health Connect/인바디/AR 연동 즉시 구현
- 모든 화면 전면 개편
- 카드/배경/몬스터 아트 전체 재생성

먼저 핵심 루프가 문서와 작은 UI 변경으로 설득되는지 확인한다.
