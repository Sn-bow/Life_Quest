## 1. 절대 규칙

### 1-1. 도형 임시 비주얼 금지

정식 배포 대상 게임 화면에서 금지:

- 플레이어를 `CustomPainter`의 사각형/원/선 조합으로 표현
- 몬스터를 `CustomPainter`의 사각형/원/선 조합으로 표현
- 카드 프레임을 Container/border/gradient만으로 대체
- 공격/방어/마법/사망 이펙트를 선, 원, 사각형만으로 최종 처리
- 이미 존재하는 PNG를 무시하고 코드 도형을 새로 만드는 행위

허용:

- 충돌 영역, 선택 범위, 디버그 표시
- 짧은 보조 파티클. 단, 주 이펙트는 이미지/스프라이트/시트/고품질 생성 에셋이어야 한다.
- 에셋 로딩 실패 시 크래시 방지용 fallback. 단, fallback은 배포 화면에서 보이면 안 된다.

### 1-2. 에셋 우선순위

1. 프로젝트에 이미 존재하는 에셋
2. 이전 Codex/Claude 세션에서 생성된 에셋
3. 공식 자료를 조사한 뒤 새로 생성한 에셋
4. 라이선스가 명확한 외부 에셋
5. 임시 placeholder. 단, 작업 로그에 제거 예정일과 대체 경로를 남긴다.

### 1-3. 작업 착수 전 조사

게임 기능 작업 전 조사해야 할 것:

- 관련 화면의 현재 코드와 에셋 경로
- 기존 문서와 공유 작업 로그
- Slay the Spire 2 또는 동급 카드 로그라이크의 해당 UI 패턴
- 모바일 터치/가독성 제약
- Flutter/Flame에서 해당 구현 방식의 안정성

## 2. 현재 게임 상태 진단

### 이미 갖춘 것

- 카드 데이터 207장
- 카드 카테고리: Attack, Magic, Defense, Tactical
- 희귀도: Common, Uncommon, Rare, Legendary
- 전투 상태 로직: `lib/state/card_combat_state.dart`
- 던전 상태/맵/상점/휴식/이벤트/결과/무한 타워
- 몬스터 데이터와 실제 몬스터 PNG: `assets/images/monsters/`
- 배경 PNG 5종: `assets/images/backgrounds/`
- 카드 프레임 PNG 4종: `assets/images/cards/card_frame_*.png`
- 유물 PNG 다수: `assets/images/game/relics/`
- 전투 SFX 다수: `assets/sounds/sfx/`, `assets/sounds/game/`

### 부족한 것

- `assets/images/game/player/`, `game/monsters/`, `game/backgrounds/`, `game/effects/`, `game/cards/frames/`가 실질적으로 비어 있다.
- `PlayerBattleSprite`와 `MonsterBattleSprite`가 `CustomPainter` 도형 기반이다.
- `BattleGame` 이펙트가 Canvas 선/원/사각형 기반이다.
- 카드 UI가 전투/상점/보상/컬렉션에서 일관된 에셋 기반 컴포넌트로 통합되지 않았다.
- 카드별 고유 일러스트가 없다.
- 전투 레이아웃이 카드 로그라이크의 정보 위계를 충분히 따르지 않는다.
- 밸런스 플레이테스트 기록이 없다.

## 3. 목표 경험

### 전투 첫인상

사용자가 전투 화면에 들어오면 즉시 보여야 하는 것:

- 현재 존 분위기가 배경으로 보인다.
- 적은 실제 몬스터 이미지로 보인다.
- 플레이어는 단순 도형이 아니라 일상 RPG 주인공 콘셉트의 일러스트/스프라이트로 보인다.
- 적 의도, HP, 블록, 상태이상이 한눈에 읽힌다.
- 하단 카드 패가 카드 게임답게 보인다.
- 카드의 비용, 타입, 희귀도, 핵심 효과가 1초 안에 읽힌다.

### 카드 플레이 감각

- 사용 가능한 카드는 명확히 떠 보인다.
- 비용 부족 카드는 흐려지되 텍스트는 읽힌다.
- 카드를 내면 사운드, 이동, 타격, 피해 숫자, 적 hit feedback이 순서대로 발생한다.
- 전투 결과는 카드 보상/골드/유물 획득으로 바로 이어진다.

### 반복 플레이 감각

- 같은 존에서도 적 조합, 이벤트, 보상 선택이 달라진다.
- 카드 보상은 무작위지만 빌드 방향을 만들 수 있다.
- 카드 제거/업그레이드/상점/휴식이 의미 있는 선택이 된다.

### 4-3. 몬스터 에셋

현재 `assets/images/monsters/`의 PNG를 우선 사용한다.

작업:

1. `MonsterBattleSprite`의 `CustomPainter`를 `Image.asset(monster.spritePath)` 기반으로 교체한다.
2. hit flash, 흔들림, 사망 fade/shatter는 이미지 위 오버레이로 처리한다.
3. 몬스터 PNG가 없는 경우만 placeholder를 쓰고, 로그에 누락을 남긴다.
4. `assets/images/game/monsters/`로 복사할지 기존 경로를 유지할지 결정한다. 중복을 피하려면 기존 `assets/images/monsters/` 유지가 우선이다.

완료 기준:

- 전투 화면의 모든 적이 실제 PNG로 표시
- 도형 몬스터 painter 제거 또는 디버그 전용 격리
- 몬스터 spritePath 에셋 존재 테스트 추가

### 4-4. 배경 에셋

- `bg_zone1_meadow.png`
- `bg_zone2_dark_forest.png`
- `bg_zone3_stone_castle.png`

### 4-5. 카드 에셋

현재 카드 프레임:

- `assets/images/cards/card_frame_attack.png`
- `card_frame_defense.png`
- `card_frame_magic.png`
- `card_frame_tactical.png`

문제:

- STS2식 카드 구조는 타입별로 완전히 다른 그림을 만드는 것이 아니라, 정보 배치와 프레임 언어가 일관되어야 한다.
- 현재 전투/상점/보상/컬렉션의 카드 렌더링이 분산되어 있다.

작업:

1. 공통 카드 컴포넌트 `SoulDeckCardView`를 만든다.
2. 같은 컴포넌트를 전투 패, 보상 선택, 카드팩, 컬렉션, 상점, 휴식에서 사용한다.
3. 카드 영역:
   - 좌상단 비용 오브
   - 상단 이름 배너
   - 중앙 일러스트 슬롯
   - 하단 효과 텍스트
   - 희귀도 테두리/광택
   - 타입 아이콘
4. 카드별 일러스트가 없으면 카테고리 아이콘을 확대해서 넣지 않는다. 누락 상태를 명확히 기록하고 우선 30장부터 생성한다.

카드 일러스트 우선순위:

- 스타터 10장
- 각 카테고리 Common 핵심 8장
- 빌드 핵심 Uncommon/Rare 12장
- 총 30장 1차 완료 후 확장

## 5. 전투 UX 개편

### 5-1. 레이아웃

모바일 세로 기준:

- 상단: 전투 정보, 턴, 드로우/버림/소멸 카운트
- 상중단: 적 영역. 몬스터 이미지, 의도, HP, 상태이상
- 중단: 플레이어 영역. 플레이어 이미지, HP, 블록, 에너지
- 하단: 카드 패. 3~5장 가시, 좌우 스크롤 가능
- 우하단: 턴 종료 버튼. 카드와 겹치지 않음

금지:

- 적을 카드처럼 프레임 안에 가두는 구성
- 턴 종료 버튼이 카드 패와 같은 위계로 섞이는 구성
- HP/에너지/의도가 작은 텍스트만으로 표시되는 구성

### 5-2. 적 의도

작업:

- 의도 타입별 아이콘: 공격, 다중공격, 방어, 버프, 디버프, 알 수 없음
- 데미지 수치와 횟수를 아이콘 옆에 표시
- 다음 행동 변화가 있는 경우 작은 보조 표시

완료 기준:

- 전투 중 플레이어가 적 턴 피해를 예측할 수 있음
- 다중 적 전투에서 각 적의 의도가 독립적으로 보임

### 5-3. 카드 상호작용

작업:

- 카드 탭: 상세 보기
- 카드 드래그 또는 탭 후 대상 선택: 공격 카드
- 대상 없는 카드: 즉시 발동
- 비용 부족: 카드 사용 불가 피드백
- 장시간 누름: 키워드 설명

완료 기준:

- 한 손 조작 가능
- 작은 화면에서도 카드 이름/비용/핵심 효과가 읽힘
- 잘못 탭했을 때 취소 가능

### 5-4. 피드백

작업:

- 카드 사용 사운드: 카테고리별 분리
- 공격/방어/마법/전술 이펙트: 에셋 기반
- 적 피격: shake + flash + damage text
- 플레이어 피격: 화면 shake는 큰 피해에만 제한
- 승리/패배: 짧고 명확한 결과 연출

## 6. 던전/보상/진행

### 6-1. 던전 맵

작업:

- 노드 타입 시각 언어 정리: 전투, 엘리트, 보스, 이벤트, 상점, 휴식, 보물
- 현재 위치, 선택 가능 경로, 잠긴 경로를 명확히 구분
- 노드 아이콘은 단순 Material Icon만으로 끝내지 않고 게임 UI 톤에 맞춘 에셋 또는 커스텀 아이콘으로 정리

### 6-2. 보상

작업:

- 전투 보상 화면에서 카드 3장 선택의 기대감을 강화
- 카드 보상은 희귀도/카테고리/빌드 시너지가 읽히게 표시
- 유물 획득 시 아이콘, 이름, 효과, 즉시 적용 여부를 보여준다.
- 스킵 선택은 명확하지만 과도하게 유도하지 않는다.

### 6-3. 상점/휴식/이벤트

작업:

- 상점: 카드/유물/제거의 가격 곡선 재검토
- 휴식: 회복 vs 업그레이드 선택 가치 검토
- 이벤트: 선택지 결과의 위험/보상 문구를 명확히 한다.

## 7. 밸런스 계획

### 7-1. 데이터 분석

산출물:

분석 항목:

- 카드별 기대 피해/방어/드로우/에너지 효율
- 희귀도별 평균 효율

- 유물 중첩 시 폭발 위험
- 존별 몬스터 HP/공격 곡선
- 보스 클리어 요구 턴 수

### 7-2. 플레이테스트

최소 시나리오:

- 스타터 덱만으로 Zone 1 보스까지 5회
- 공격 중심 덱 5회
- 방어 중심 덱 5회
- 마법 중심 덱 5회
- 전술/드로우 중심 덱 5회
- 무한 타워 30층 3회

기록:

- 승/패
- 클리어 턴
- 남은 HP
- 가장 강한 카드/유물
- 선택하지 않은 카드 이유
- 지루했던 구간
- 이해되지 않은 효과

완료 기준:

- Zone 1 신규 사용자 예상 승률 60~80%
- Zone 2 이후 난이도 상승 체감
- 특정 카드 1장만으로 모든 전투가 해결되지 않음
- 방어만 쌓아도 무한히 버티는 구조 방지

## 9. 완료 판정

배포 가능:

- 전투 화면에서 플레이어/몬스터/카드/배경이 모두 에셋 기반
- 도형 기반 painter가 최종 게임 비주얼로 보이지 않음
- 카드 30장 이상 고유 일러스트 적용 또는 명확한 고품질 대체 일러스트 적용
- 5개 존/보스/무한 타워 주요 루프 QA 완료
- 카드/몬스터/유물 에셋 누락 테스트 통과
- 실제 기기에서 30분 플레이 중 크래시 없음

배포 불가:

- 플레이어 또는 몬스터가 사각형/원/선 조합으로 보임
- 카드가 색상 박스와 텍스트 위주로 보임
- 전투 루프가 이해되지 않아 튜토리얼 없이 진행 불가
- 밸런스 기록 없이 수치가 감으로 조정됨
