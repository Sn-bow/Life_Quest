# Life Quest Core Loop UX Detailed Plan - 2026-05-16

## 목적

Life Quest의 리메이크 핵심 루프를 상세 설계한다. 목표는 사용자가 "현실 행동이 게임 성장으로 바뀐다"는 것을 첫 30초 안에 이해하고, 퀘스트 완료 후 던전/칭호/보상에서 그 결과를 체감하게 만드는 것이다.

## 핵심 문장

```text
오늘의 현실 퀘스트를 완료하면 나의 성향과 스탯이 성장하고,
그 성장이 오늘의 던전 보정과 칭호/이벤트 해금으로 이어진다.
```

이 문장으로 설명되지 않는 기능은 전면에 두지 않는다.

---

## Core Loop

### 1. 행동

사용자는 현실 행동을 퀘스트로 만든다.

예:
- 운동 30분
- 책 20쪽 읽기
- 프로젝트 작업 1시간
- 방 정리
- 물 2L 마시기
- 연락하기

### 2. 완료

퀘스트 완료 시 앱은 즉시 보상을 보여준다.

필수 표시:
- XP
- 골드
- 성장 스탯
- 오늘 던전 보정
- 다음 칭호 진행도

### 3. 성장

완료한 퀘스트의 카테고리와 난이도에 따라 성장값이 쌓인다.

성장은 단순 숫자가 아니라 "오늘의 성향"과 "던전 효과"로 번역되어야 한다.

### 4. 모험

오늘 완료한 현실 행동이 던전에서 다음 방식으로 체감된다.

- 전투 시작 보정
- 카드 보상 확률
- 이벤트 선택지
- 상점 할인
- 휴식 회복량
- 칭호 전용 이벤트

### 5. 회귀

던전 결과는 다시 현실 행동을 유도한다.

예:
- `지혜 퀘스트를 1개 더 완료하면 마법 카드 보상 확률 +5%`
- `건강 퀘스트를 완료하면 다음 전투 HP +5`
- `3일 연속 완료하면 루틴 입문자 칭호 해금`

## 데이터 모델 초안

### QuestCategory

현재 카테고리를 아래 4개 축으로 수렴한다.

| 카테고리 | 의미 | 대표 행동 | 기본 스탯 |
|---|---|---|---|
| strength | 실행력/몸/돌파 | 운동, 청소, 힘든 일 처리 | 힘 |
| wisdom | 학습/분석/창작 | 공부, 독서, 글쓰기, 코딩 | 지혜 |
| health | 회복/생활 리듬 | 수면, 식사, 물, 산책, 휴식 | 건강 |
| charisma | 관계/표현/외부 연결 | 연락, 발표, 공유, 피드백 | 매력 |

### GrowthDelta

퀘스트 완료 1회가 만드는 성장 변화.

```dart
class GrowthDelta {
  final double strength;
  final double wisdom;
  final double health;
  final double charisma;
  final int xp;
  final int gold;
}
```

### DailyModifier

오늘 완료한 행동이 오늘 던전에 주는 보정.

```dart
class DailyModifier {
  final int combatHpBonus;
  final int attackDamageBonus;
  final int startingGoldBonus;
  final int startingEpBonus;
  final int firstTurnDrawBonus;
  final double defenseCardWeightBonus;
  final double magicCardWeightBonus;
  final double eventOptionBonusChance;
}
```

### TitleUnlockProgress

칭호는 단순 수집물이 아니라 이벤트/보정 unlock 조건이다.

```dart
class TitleUnlockProgress {
  final String titleId;
  final int current;
  final int required;
  final List<String> unlockedEventIds;
  final List<String> unlockedModifierIds;
}
```

## 성장 변환 규칙

### 난이도 계수

| 난이도 | XP | 골드 | 스탯 계수 | 설명 |
|---|---:|---:|---:|---|
| easy | 10 | 5 | 0.2 | 작고 가벼운 행동 |
| normal | 20 | 10 | 0.4 | 기본 행동 |
| hard | 40 | 20 | 0.8 | 부담 있는 행동 |
| veryHard | 70 | 35 | 1.2 | 장기 집중/강한 노력 |

### 카테고리별 성장

| 카테고리 | 주 스탯 | 보조 스탯 | 던전 보정 |
|---|---|---|---|
| strength | 힘 +100% | 건강 +30% | 공격 카드 피해, 시작 EP |
| wisdom | 지혜 +100% | 매력 +20% | 마법 카드 보상, 첫 턴 드로우 |
| health | 건강 +100% | 힘 +20% | 전투 HP, 회복/방어 |
| charisma | 매력 +100% | 지혜 +20% | 이벤트 선택지, 상점 할인 |

### 예시 계산

운동 30분, normal, strength:

```text
XP +20
Gold +10
힘 +0.4
건강 +0.12
오늘 던전: 공격 카드 피해 +1 또는 시작 EP 게이지 +1 진행
칭호: 꾸준한 전사 +1
```

책 20쪽 읽기, normal, wisdom:

```text
XP +20
Gold +10
지혜 +0.4
매력 +0.08
오늘 던전: 첫 턴 드로우 +1 진행 또는 마법 카드 보상 확률 +5%
칭호: 지식 수집가 +1
```

## 오늘의 모험 화면 상세

### 화면 목적

첫 화면에서 아래를 한 번에 보여준다.

```text
오늘 내가 무엇을 하면
어떤 성장과 던전 보정이 생기고
다음에 무엇이 열리는가
```

### 정보 위계

1. 오늘의 한 줄 상태
2. 오늘의 추천 행동
3. 오늘의 퀘스트
4. 예상 성장
5. 오늘의 던전 보정
6. 다음 칭호/이벤트
7. 집중 타이머 진입

### UI 블록

#### A. 오늘의 모험 헤더

필수 텍스트:

```text
오늘의 모험
현실 퀘스트를 완료하면 오늘 던전에서 힘으로 바뀝니다
```

표시:
- 완료 퀘스트 수
- 오늘 얻은 XP
- 오늘 얻은 골드
- 오늘 주 성향

#### B. 추천 행동

규칙:
- 한 번에 1개만 추천
- 추천 이유를 1줄로 설명
- 바로 시작 가능한 CTA 제공

예:

```text
추천: 25분 집중
이유: 지혜 퀘스트가 부족해 오늘 마법 보정이 비어 있습니다
```

#### C. 오늘 퀘스트 리스트

각 항목에 표시:
- 이름
- 카테고리 아이콘
- 완료 보상
- 던전 효과 미리보기

예:

```text
운동 30분
XP +20 / 골드 +10 / 힘 +0.4
오늘 던전: 공격 피해 +1 진행
```

#### D. 오늘 던전 보정

표시 방식:

```text
오늘 적용 중
- 전투 HP +5
- 마법 카드 등장률 +5%
- 이벤트 선택지 +1 준비 중
```

CTA:
- `보정 적용하고 던전 시작`

#### E. 다음 칭호/이벤트

예:

```text
다음 칭호: 루틴 입문자
3일 연속 완료까지 1일 남음
해금: 첫 캠프 이벤트
```

## 던전 연결 규칙

### 던전 시작 전

던전 홈에서 오늘의 보정을 명시한다.

필수:
- 어떤 현실 행동 때문에 생긴 보정인지 표시
- 보정이 이번 런에만 적용되는지 표시
- 던전 시작 전 취소/확인 가능

예:

```text
오늘의 현실 행동 보정
운동 30분 완료 -> 전투 HP +5
독서 완료 -> 첫 턴 카드 +1
```

### 전투 화면

과밀하게 설명하지 않는다.

표시:
- HP/EP 옆 작은 `오늘 보정` 아이콘
- 탭하면 상세 표시

예:

```text
오늘 보정
건강 퀘스트: 전투 HP +5
지혜 퀘스트: 첫 턴 드로우 +1
```

### 보상 화면

카드 보상에 이유를 붙인다.

예:

```text
지혜 퀘스트 덕분에 마법 카드 등장률이 올랐습니다
```

### 이벤트 화면

칭호/스탯 조건 선택지를 추가한다.

예:

```text
[지혜 5+] 고대 문자를 해석한다
[루틴 입문자] 안전한 야영지를 찾는다
```

## 칭호-이벤트 매핑

### 출시 1차 칭호 후보

| 칭호 | 조건 | 해금 효과 | 이벤트 |
|---|---|---|---|
| 루틴 입문자 | 3일 연속 퀘스트 완료 | 캠프 이벤트 해금 | 첫 캠프 |
| 꾸준한 전사 | strength 퀘스트 10회 | 공격 카드 보상 +5% | 훈련장 |
| 지식 수집가 | wisdom 퀘스트 10회 | 마법 카드 보상 +5% | 고대 서재 |
| 회복의 관리자 | health 퀘스트 10회 | 휴식 회복량 +10% | 온천 |
| 연결의 중재자 | charisma 퀘스트 10회 | 이벤트 선택지 +1 | 여행 상인 |
| 새벽 탐험가 | 오전 퀘스트 5회 | 하루 첫 퀘스트 보너스 | 새벽 길목 |
| 집중 견습생 | 타이머 5회 완료 | 첫 턴 드로우 +1 조건 | 조용한 방 |
| 정리의 장인 | 정리/청소 7회 | 방어 카드 보상 +5% | 정돈된 창고 |
| 작은 승리 수집가 | easy 퀘스트 20회 | 골드 +5% | 작은 보물상자 |
| 큰 산을 넘은 자 | veryHard 3회 | 희귀 카드 보상 확률 +3% | 산 정상 |

### 체크리스트

- [ ] 기존 `title_database.dart` 칭호 목록 조사
- [ ] 출시 1차 칭호 10~20개 선정
- [ ] 각 칭호에 조건 1개, 효과 1개, 이벤트 1개 매핑
- [ ] 이벤트 DB와 연결 가능한 구조 설계
- [ ] UI에서 다음 칭호까지 남은 조건 표시

## 추천 행동 규칙

### 목표

AI 없이도 납득 가능한 추천을 만든다.

### 우선순위

1. 오늘 아직 완료한 퀘스트가 없으면 가장 쉬운 퀘스트 추천
2. 가장 낮은 스탯 카테고리 추천
3. 다음 칭호에 가까운 행동 추천
4. 던전 보정이 비어 있는 카테고리 추천
5. 사용자가 자주 완료하는 카테고리 추천

### 예시 문구

```text
오늘은 건강 보정이 비어 있습니다. 산책 10분을 완료하면 전투 HP +5를 받을 수 있습니다.
```

```text
루틴 입문자까지 1일 남았습니다. 아무 일일 퀘스트나 하나 완료해보세요.
```

```text
지혜가 가장 낮습니다. 25분 집중을 완료하면 첫 턴 드로우 보정이 열립니다.
```

## 수치 밸런스 1차 기준

출시 1차는 보수적으로 잡는다.

| 효과 | 최소 | 최대 | 이유 |
|---|---:|---:|---|
| 전투 HP 보정 | +3 | +15 | 너무 크면 던전 밸런스 붕괴 |
| 공격 피해 보정 | +1 | +3 | 카드 수치가 작아 영향 큼 |
| 첫 턴 드로우 | +0 | +1 | +2 이상은 과함 |
| 카드 보상 확률 | +3% | +10% | 체감은 되지만 지배적이면 안 됨 |
| 상점 할인 | 5% | 15% | 골드 경제 보호 |
| 휴식 회복량 | +5% | +20% | 체감용 |
| 이벤트 선택지 | +0 | +1 | UI 복잡도 제한 |

## 완료 조건

- 현실 행동이 어떤 성장으로 바뀌는지 표로 설명된다.
- 성장값이 어떤 던전 보정으로 바뀌는지 표로 설명된다.
- 오늘의 모험 화면에 들어갈 정보 위계가 확정된다.
- 칭호가 최소 10개 이상 이벤트/효과와 연결된다.
- 구현 체크리스트가 모델/상태/UI/게임/검증 단위로 나뉜다.
