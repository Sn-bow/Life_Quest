# Threads Core Loop Feedback Addendum - 2026-05-17

## 요약

추가 피드백의 핵심은 "던전 자체"가 아니라 "현실 행동과 상태 체크 자체가 게임 경험이 되어야 한다"는 점이다. Life Quest는 던전을 고집하는 앱이 아니라, 현실 행동 데이터를 RPG식 성장, 경쟁, 보상, 다음 행동 추천으로 번역하는 앱이어야 한다.

## 받아들일 결정

1. 던전은 핵심 정체성이 아니라 성장 체감용 보조 루프다.
2. 핵심 루프는 `현실 행동 기록 -> 상태 변화 -> 성장/칭호/보상 -> 다음 행동 추천`이다.
3. 사용자가 앱을 여는 첫 화면은 "오늘 무엇을 하면 어떤 변화가 생기는지"를 보여줘야 한다.
4. 장기적으로 경쟁/랭킹/그룹 챌린지는 가능성이 있지만, 1차 리메이크에서는 실시간 PVP가 아니라 주간 기록 기반 경쟁만 후보로 둔다.
5. AR, 모션 인식, Health Connect 같은 자동 기록은 즉시 구현하지 않는다. 먼저 앱의 수동 기록 루프가 충분히 명확한지 검증한다.

## 계획서 통합 방식

기존 계획서의 방향을 아래처럼 조정한다.

```text
이전 중심:
현실 퀘스트 -> 성장 -> 오늘 던전 보정 -> 던전에서 체감

수정된 중심:
현실 행동/상태 체크 -> 성장/상태 변화 -> 오늘의 추천 행동/칭호/보상 -> 선택적으로 던전에서 체감
```

따라서 "오늘의 모험"은 단순 던전 보정 패널이 아니라 오늘의 상태판이어야 한다.

- 오늘 완료한 현실 행동
- 오늘 부족한 행동 카테고리
- 오늘 얻은 성장/XP/골드
- 오늘 열리는 보상/칭호/이벤트
- 선택적으로 던전에 적용되는 보정
- 다음 추천 행동 1개

## 기능 우선순위 변경

### P0

- 오늘의 모험을 현실 행동 상태판으로 확장
- 현실 행동 카테고리와 성장/보정 매핑을 실제 수치에 적용
- 던전 시작 시 오늘 보정 스냅샷을 저장하고 전투에 일부 반영
- 앱 첫 진입 메시지를 "던전 게임"이 아니라 "현실 행동을 성장으로 번역"으로 바꿈

### P1

- 칭호가 이벤트/추천 행동/보상 조건으로 작동하게 연결
- 주간 요약, 주간 기록 기반 랭킹/챌린지 설계
- 현실 보상 상점과 오늘의 행동 추천 연결

### P2

- Health Connect, Google Fit, AR, 모션 인식
- 자동 식단/칼로리/수면 분석
- 실시간 PVP 또는 강한 경쟁 구조

## 현재 구현 반영

2026-05-17 작업부터 `DailyModifier`를 던전 시작 스냅샷에 저장하고, HP/골드/공격 보정/첫 턴 드로우에 실제 적용한다. 이것은 "현실 행동이 선택적으로 던전에서 체감된다"는 연결의 첫 단계다.
